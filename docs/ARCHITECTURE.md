# System Architecture

## 1. 전체 흐름

공식 재무정보 입력
→ 데이터 형식 및 출처 검증
→ Python 기반 관계 계산
→ 사전 정의 규칙 판정
→ Red Flag와 Monitoring Signal 분류
→ 위험군별 중복 제거
→ 경영 관점과 감사 관점 해석
→ 인터뷰 질문 및 증빙 요청
→ Trusted Output 생성

## 2. 결정론적 계산 영역

Python과 pandas가 담당하는 영역은 다음과 같습니다.

- 전년 대비 증감률
- 매출액 대비 계정 비율
- 영업이익률과 순이익률
- 영업현금흐름률
- 영업권과 무형자산의 총자산 대비 비중
- 총자산회전율 및 근사 ROA
- 규칙 충족 여부
- 검토 우선순위 점수
- 위험군별 중복 제거

이 계산은 동일한 입력에 대해 동일한 결과를 반환합니다.

## 3. 해석 및 질문 영역

다음 결과는 계산 결과를 바탕으로 한 추가 검토 가설입니다.

- 가능한 정상적 설명
- 가능한 위험 해석
- 경영진 인터뷰 질문
- 요청할 증빙
- 후속 검토 행동

현재 MVP에서는 설계자가 직접 작성했으며,
향후 생성형 AI API 연결 대상으로 구분합니다.

## 4. Trusted AI 설계 원칙

- 사실과 가설을 분리합니다.
- 규칙 충족과 최종 판단을 분리합니다.
- 규칙 미충족 결과도 공개합니다.
- 데이터 완전성과 근거 강도를 표시합니다.
- 우선순위 점수를 오류확률로 표현하지 않습니다.
- 외부 검증 결과를 정확도로 과장하지 않습니다.
- 생성형 AI는 계산 결과를 변경하지 못하도록 분리합니다.

## 5. 향후 구조

재무데이터 수집 계층
→ 데이터 검증 계층
→ 관계형 분석 엔진
→ 업종별 규칙 모듈
→ 근거 문서 검색 계층
→ 생성형 AI 질문 생성 계층
→ 사용자 검토 및 승인
→ 보고서와 감사문서 출력
